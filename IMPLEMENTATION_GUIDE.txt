# Guía paso a paso – Implementación de la Onion Architecture

## 1️⃣ Preparación del proyecto
- `go mod init companies-api`
- Instalar dependencias: Gin, GORM, PostgreSQL driver, SQLite driver, JWT, validator, zap.
- Crear `.gitignore` (incluye `.env`, binaries, `vendor`).

## 2️⃣ Capa de Dominio (Domain)
- **Entidades** en `domain/entities/` (Compania, Empleado, Usuario) sin tags de GORM.
- **Interfaces de repositorio** en `domain/interfaces/` (`ICompaniaRepository`, `IEmpleadoRepository`, `IUsuarioRepository`).
- **Interface UnitOfWork** en `domain/interfaces/unit_of_work.go` que expone los repositorios y métodos Begin/Commit/Rollback.

## 3️⃣ Capa de Aplicación (Application)
- **DTOs** con tags `validate` en `application/dtos/`.
- **Servicios** (`CompaniaService`, `EmpleadoService`, `AuthService`) que reciben un `IUnitOfWork` y un `validator`. Cada método:
  1. Valida el DTO.
  2. Llama `uow.Begin()`.
  3. Ejecuta operaciones a través de los repositorios (`uow.Companias().Create(...)`).
  4. `uow.Commit()` o `uow.Rollback()` según el error.
- **UnitOfWork Implementation** en `application/infrastructure/unit_of_work/` usando GORM (`*gorm.DB`). Guarda la transacción activa (`tx`).

## 4️⃣ Capa de Infraestructura (Infrastructure)
- **Conexión DB** (`infrastructure/database/database.go`) lee variables de entorno (`DB_HOST`, `DB_USER`, …) y crea `*gorm.DB`.
- **Repositorios concretos** en `infrastructure/repositories/` que implementan las interfaces del dominio usando GORM (`db.WithContext(ctx).Create(... )`). No gestionan transacciones, usan la `*gorm.DB` proporcionada por el UnitOfWork.
- **Migraciones** (`AutoMigrate`) al iniciar la aplicación (`database/migrate.go`).

## 5️⃣ Capa de Presentación / API (API)
- **Middlewares** (`api/middlewares/`):
  - `AuthMiddleware` → valida JWT, guarda claims (`user_id`, `rol`, `user_compania_id`).
  - `RequireRole` → verifica rol (ADMIN / USUARIO).
  - `EsPropietarioDeCompania` → política de ownership.
  - `LoggerMiddleware` → registro estructurado con zap.
- **Controladores** (`api/controllers/`) reciben los servicios mediante inyección de dependencias y traducen peticiones HTTP a llamadas de servicio.
- **Rutas** (`api/routes/routes.go`) configuran los endpoints y aplican los middlewares correspondientes.

## 6️⃣ Bootstrap (main.go)
1. Cargar `.env`.
2. Inicializar logger.
3. Crear la conexión DB.
4. Instanciar UnitOfWork (`uow := infrastructure.NewUnitOfWork(db)`).
5. Crear repositorios (`compRepo := infrastructure.NewCompaniaRepository(db)`, etc.).
6. Crear servicios (`compService := application.NewCompaniaService(uow, validator.New())`).
7. Crear controladores (`compCtrl := api.NewCompaniaController(compService)`).
8. Configurar el router Gin y registrar rutas.
9. Ejecutar `router.Run(":" + os.Getenv("PORT"))`.

## 7️⃣ Testing
- **Base de datos en memoria** (`sqlite :memory:`) en los tests.
- Instanciar UnitOfWork con esa DB y pasar a los servicios.
- Pruebas unitarias de cada servicio (validaciones, transacciones, rollback).
- Pruebas de integración de endpoints con `httptest` (`router_test.go`).
- Verificar códigos HTTP correctos (200, 201, 204, 400, 404, 422, 403).

## 8️⃣ Documentación
- Actualizar `README.md` con arquitectura, tecnología, variables de entorno, cómo ejecutar y probar.
- Añadir secciones extra (`README_ADDITIONS.md`) con async, tabla comparativa, evidencia de prompts IA y sustentación.

---
**Resultado:** Con estos archivos y estructura el proyecto sigue la Onion Architecture, mantiene bajo acoplamiento entre capas, permite pruebas aisladas y soporta transacciones atómicas y seguridad basada en JWT.
